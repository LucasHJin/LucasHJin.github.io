Portfolio

Competitive Programming Problems

A collection of all the competitive programming problems I have solved. It covers a wide variety of DSA problems, ranging from DP to graph theory. It contains problems from various sources including AtCoder, CSES, DMOJ and CodeForces and most are solved with C++.

Lips Lips Revolution

Lips Lips Revolution won at Hack the North 11 for MLH's Best Use of MongoDB Atlas and it is a social platform where you compete against others by lip syncing to short song clips. It was built with Next.js as the full stack framework, Tailwind CSS for styling and MongoDB for the database. To read the player's lips, Symphonic Labs' API 'Mamo' was used (which we actually reverse engineered with Wireshark and Postman) and to compare the scores, an OpenAI embedding model was used.

ConvoAI

Designed for improved programming interview performance, ConvoAI is the extra boost you need to land your job! Using Google's speech recognition model along with OpenAI's machine learning models, it recognizes when a question is asked and generates a response rapidly. It can even personalize responses based on your resume. Alongside these external pieces, it is made with HTML, CSS and JavaScript as well as Python and Flask.

StudySphere

Made for UOttaHack 6, StudySphere helps students study with peers. It provides the ability to ask questions, answer questions and search for them, essentially acting like a Reddit/Discord for studying. The front end is made up of HTML, CSS and JavaScript and the back end was made with Flask, Python and SQLAlchemy.

FRC Team 8729 Attendance Bot

This bot tracks the attendance of the 90+ members on the SYRC robotics team. It uses integrates the Discord API with the Sheets API using Python and OOP to create a modular and reusable bot with clockin/clockout functionality. Additionally, it utilizes an SQLite database for further data storage.

LazyReps

Lazy Reps won solo at LazyHacks in the Home track category; it is an application to automate tracking the reps of a set. To do this, it uses the TensorFlow Movenet Single Pose Lightning Model to measure joints and determine the angle of flexion or contraction. It was developed with React.js for the frontend and FastAPI for RESTful API calls in the backend to incorporate writing the reps to a google sheet.

STEM-E Volunteer App

As an intern for the nonprofit STEM-E, I am designing a web application to automate the process for 1000+ of volunteers. I am following agile methodology to ensure efficient collaboration and am using React.js for the frontend framework, Firebase as a BaaS (specifically for its database) and Next.js and Express for API calls to write to a google sheet.

Sparking H2O - FRC Team 8729

During the 2023-2024 season, I was a core programmer for FRC team 8729, working on implementing robot vision and april tag recognition with command based programming. During the 2024-2025 season, I will be a software lead, teaching new software members both command based programming and electrical.

Zenith: Workout App

Made as a passion project, Zenith App is an IOS-only app to track and visualize progress in the gym. It allows users to put in their sets, reps and comments for an exercise, watch tutorials for unknown exercises and much more. It is made with Swift and SwiftUI and uses Firebase for user authentication and a document database.
